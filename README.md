## Projects and areas I care about

### 🤝 Quadro Sociale / GPC-style platforms

I'm building and researching systems for managing social-service workflows: people, access records, documents, operators, structures, permissions, reminders and reporting.

The goal is to create software that is useful in real organizations, especially where data protection, traceability and usability matter.

Key themes:

- Multi-tenant architecture
- Role-based access control
- Firebase Auth and session cookies
- Firestore security rules
- GDPR-aware document storage
- Per-structure permissions
- Auditability and future reporting

## How I think about software

I like software that is:

- *Useful* before it is flashy
- *Secure by design*, not patched later
- *Understandable* by the people who need to use it
- *Scalable enough*, without unnecessary complexity
- *Respectful of data*, especially when dealing with sensitive contexts

I'm especially interested in products where engineering, operations, privacy and real human workflows meet.
